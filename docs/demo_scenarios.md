## 📋 Scénario 2: Segmentation et Analyse Client

### 🎬 Contexte de la Démonstration
*"L'équipe marketing veut personnaliser ses campagnes selon les segments clients."*

#### 🔹 Agent Snowflake (Détail)
**Question** : "Quels sont mes clients VIP (segment Premium) qui n'ont pas commandé depuis 2 mois ?"

**Requête générée** :
```sql
SELECT 
    c.CUSTOMER_ID,
    c.FIRST_NAME || ' ' || c.LAST_NAME as CLIENT,
    c.EMAIL,
    c.CITY,
    SUM(o.TOTAL_AMOUNT) as CA_TOTAL,
    COUNT(DISTINCT o.ORDER_ID) as NB_COMMANDES,
    MAX(o.ORDER_DATE) as DERNIERE_COMMANDE,
    DATEDIFF(DAY, MAX(o.ORDER_DATE), CURRENT_DATE()) as JOURS_INACTIVITE
FROM RETAIL_DEMO.MAIN.CUSTOMERS c
JOIN RETAIL_DEMO.MAIN.ORDERS o ON c.CUSTOMER_ID = o.CUSTOMER_ID
WHERE c.CUSTOMER_SEGMENT = 'Premium'
  AND o.STATUS != 'Cancelled'
GROUP BY c.CUSTOMER_ID, c.FIRST_NAME, c.LAST_NAME, c.EMAIL, c.CITY
HAVING MAX(o.ORDER_DATE) < DATEADD(MONTH, -2, CURRENT_DATE())
  AND SUM(o.TOTAL_AMOUNT) > 1000
ORDER BY CA_TOTAL DESC;
```

**Action recommandée** :
- Liste de 50+ clients Premium à risque
- Campagne de réactivation personnalisée
- Offres spéciales selon l'historique d'achat

## 📋 Scénario 3: Performance des Magasins

### 🎬 Contexte de la Démonstration  
*"Le directeur régional veut optimiser la performance de ses points de vente."*

#### 🔹 Agent Snowflake (Détail)
**Question** : "Analyse détaillée du magasin Paris-Centre : top produits, profil clients, heures de pointe"

**Requêtes multiples** :
```sql
-- Performance globale du magasin
SELECT 
    s.NAME as MAGASIN,
    COUNT(DISTINCT o.ORDER_ID) as TOTAL_COMMANDES,
    COUNT(DISTINCT o.CUSTOMER_ID) as CLIENTS_UNIQUES, 
    SUM(o.TOTAL_AMOUNT) as CA_TOTAL,
    AVG(o.TOTAL_AMOUNT) as PANIER_MOYEN,
    SUM(o.TOTAL_AMOUNT) / s.SURFACE_M2 as CA_PAR_M2
FROM RETAIL_DEMO.MAIN.STORES s
JOIN RETAIL_DEMO.MAIN.ORDERS o ON s.STORE_ID = o.STORE_ID
WHERE s.NAME LIKE '%Paris-Centre%'
  AND o.ORDER_DATE >= DATEADD(MONTH, -3, CURRENT_DATE())
GROUP BY s.STORE_ID, s.NAME, s.SURFACE_M2;

-- Analyse par heure de la journée
SELECT 
    EXTRACT(HOUR FROM o.ORDER_DATE) as HEURE,
    COUNT(*) as NB_COMMANDES,
    SUM(o.TOTAL_AMOUNT) as CA_HEURE,
    AVG(o.TOTAL_AMOUNT) as PANIER_MOYEN_HEURE
FROM RETAIL_DEMO.MAIN.ORDERS o
JOIN RETAIL_DEMO.MAIN.STORES s ON o.STORE_ID = s.STORE_ID
WHERE s.NAME LIKE '%Paris-Centre%'
  AND o.ORDER_DATE >= DATEADD(MONTH, -1, CURRENT_DATE())
GROUP BY EXTRACT(HOUR FROM o.ORDER_DATE)
ORDER BY HEURE;
```

## 📋 Scénario 4: Gestion des Stocks et Approvisionnement

### 🎬 Contexte de la Démonstration
*"Le responsable achats veut optimiser les niveaux de stock et éviter les ruptures."*

#### 🔹 Agent Snowflake (Détail)
**Question** : "Quels produits risquent la rupture de stock dans les 2 prochaines semaines ?"

**Requête d'analyse prédictive** :
```sql
WITH product_velocity AS (
    SELECT 
        p.PRODUCT_ID,
        p.NAME,
        p.STOCK_QUANTITY,
        AVG(oi.QUANTITY) as AVG_DAILY_SALES,
        p.STOCK_QUANTITY / NULLIF(AVG(oi.QUANTITY), 0) as DAYS_OF_STOCK
    FROM RETAIL_DEMO.MAIN.PRODUCTS p
    JOIN RETAIL_DEMO.MAIN.ORDER_ITEMS oi ON p.PRODUCT_ID = oi.PRODUCT_ID  
    JOIN RETAIL_DEMO.MAIN.ORDERS o ON oi.ORDER_ID = o.ORDER_ID
    WHERE o.ORDER_DATE >= DATEADD(DAY, -30, CURRENT_DATE())
      AND o.STATUS != 'Cancelled'
    GROUP BY p.PRODUCT_ID, p.NAME, p.STOCK_QUANTITY
)
SELECT 
    PRODUCT_ID,
    NAME as PRODUIT,
    STOCK_QUANTITY as STOCK_ACTUEL,
    ROUND(AVG_DAILY_SALES, 2) as VENTE_JOUR_MOYENNE,
    ROUND(DAYS_OF_STOCK, 1) as JOURS_STOCK_RESTANT
FROM product_velocity
WHERE DAYS_OF_STOCK <= 14
  AND AVG_DAILY_SALES > 0
ORDER BY DAYS_OF_STOCK ASC;
```

#### 🔸 Agent Fabric Data (Macro)
**Question** : "Analyse les tendances de rotation des stocks par catégorie et optimise les commandes"

**Métriques analysées** :
- **Rotation des stocks** par catégorie (turnover ratio)
- **Saisonnalité** des ventes par famille produit
- **Optimisation des niveaux de stock** selon la demande prédite
- **Coût de stockage** vs **coût de rupture**

## 📋 Scénario 5: Analyse de la Rentabilité

### 🎬 Contexte de la Démonstration
*"Le directeur financier veut identifier les leviers d'amélioration de la rentabilité."*

#### 🔹 Agent Snowflake (Détail)
**Question** : "Analyse la rentabilité par produit, client et canal de vente"

**Analyse de rentabilité** :
```sql
SELECT 
    c.NAME as CATEGORIE,
    p.BRAND,
    COUNT(DISTINCT oi.ORDER_ID) as NB_VENTES,
    SUM(oi.QUANTITY) as QTE_VENDUE,
    SUM(oi.LINE_TOTAL) as CA_BRUT,
    SUM(oi.QUANTITY * p.COST) as COUT_TOTAL,
    SUM(oi.LINE_TOTAL) - SUM(oi.QUANTITY * p.COST) as PROFIT_BRUT,
    ROUND((SUM(oi.LINE_TOTAL) - SUM(oi.QUANTITY * p.COST)) / SUM(oi.LINE_TOTAL) * 100, 2) as MARGE_PCT
FROM RETAIL_DEMO.MAIN.PRODUCTS p
JOIN RETAIL_DEMO.MAIN.CATEGORIES c ON p.CATEGORY_ID = c.CATEGORY_ID
JOIN RETAIL_DEMO.MAIN.ORDER_ITEMS oi ON p.PRODUCT_ID = oi.PRODUCT_ID
JOIN RETAIL_DEMO.MAIN.ORDERS o ON oi.ORDER_ID = o.ORDER_ID
WHERE o.STATUS = 'Delivered'
  AND o.ORDER_DATE >= DATEADD(MONTH, -6, CURRENT_DATE())
GROUP BY c.CATEGORY_ID, c.NAME, p.BRAND
HAVING SUM(oi.LINE_TOTAL) > 10000
ORDER BY PROFIT_BRUT DESC;
```

**Question** : "Dashboard de rentabilité avec KPIs et recommandations stratégiques"

**KPIs de rentabilité** :
- **Marge globale** : 32.5% (objectif: 35%)
- **EBITDA par magasin** : €45K/mois en moyenne
